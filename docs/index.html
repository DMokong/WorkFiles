Four invariants the harness protects

Everything else is replaceable. These four are load-bearing — the hooks layer is where they converge.

01

Observability

Live OpenTelemetry spans + structured tool events stream to a local or central Grafana stack as the run unfolds.

02

Auditability

A tamper-evident, hash-chained JSONL ledger of what the agent did, what prompted it, and who authorized it — sealed by AWS KMS.

03

Customisability

Per-engagement scope, tool allowlists, egress rules, and pluggable test modules — drop a file under tools/, no core changes.

04

Autonomy

Long unattended runs that respect budget / turn / per-target caps, decompose into subagents, and only escalate when an operator is required.

The runtime, in layers

L0 · spec
Engagement spec — YAML scope + NL objective + operator signature
Single source of truth. The pydantic schema rejects malformed files; the detached .sig is verified before anything runs.
L1 · runtime
Ephemeral container — scoped egress, dropped caps, non-root, read-only rootfs
No persistent state outside the mounted audit volume. Default-deny egress; the metadata endpoint is never reachable.
L2 · orchestrator
ClaudeSDKClient — agent loop + subagent decomposition
Thin. Policy lives in hooks; capability lives in tool packs. It just wires them together and hands over the objective.
L3 · policy
Hooks spine — scope check · budget gate · capture · redact · seal
PreToolUse decides; PostToolUse records. This is where the four invariants converge.
L4 · tools
In-process MCP packs — recon · web · cloud · network · whitebox · report
Each tool re-checks scope as defence-in-depth: the hook is the gate, the tool is the lock.
L5 · sinks
OTel collector (live) + signed JSONL ledger (audit)
Observability and auditability terminate here. The ledger survives the container; the operator collects it after exit.
// engagement lifecycle

From YAML to a sealed ledger

Eight stages. The operator owns the first three; the harness owns the rest and never improvises scope.

1

Author the engagement

Declare scope (targets, out-of-scope, egress), budget, tools, subagents, and the natural-language objective in one versioned file.

engagements/example.yaml
2

Sign it (detached)

Sign the exact file bytes with your SSH key. The signature lives in a sidecar — never embedded in the bytes it signs.

ssh-keygen -Y sign → .sig
3

Stage the whitebox surface

Optionally clone target repos / IaC / specs into ./targets. They mount read-only; the agent can read but never write or exfiltrate them.

gh repo clone … ./targets
4

Verify & build options

The signature is checked against authorized_signers and bound to the operator. Only then is the SDK client constructed — unsigned engagements fail closed.

redteam run …
5

Gate every tool call

PreToolUse runs the budget gate then the scope guard (deny-wins). PostToolUse hashes the raw output, redacts secrets, and emits telemetry.

permissionDecision: deny | allow
6

Tools act, in scope

The allowed packs execute — DNS, scoped HTTP, AWS reads, port probes, source scans — each re-checking scope as its own lock.

mcp__<pack>__<tool>
7

Record & report

Findings are written to SARIF on the audit volume and appended to the ledger; subagents hand off recon → analyst → exploiter.

report__write_finding
8

Seal & verify

At session end the chain head is HMAC-sealed by KMS. An auditor with verify-only IAM confirms authenticity without ever running the harness.

redteam-verify ledger seal

How to drive it

Local validation and dry-runs touch no disk and need no SDK call. A real run verifies the signature, then streams the engagement.

install & validatecopy
# editable install with dev extras
pip install -e ".[dev]"

# parse-only check — no run
redteam validate engagements/example.yaml
# → OK: ENG-2026-04-001 (4 packs, 0 MCPs)
dry-run · zero side effectscopy
# build the SDK options, write nothing
redteam run engagements/example.yaml --dry-run \
  --assets-root .
# Allowed tools: mcp__recon, mcp__web …
# MCP servers:  recon, web, whitebox, report
sign & run for realcopy
ssh-keygen -Y sign -f ~/.ssh/id_ed25519 \
  -n redteam-engagement engagements/example.yaml
redteam run engagements/example.yaml --assets-root .
# Signature OK: alice@example.com
# unsigned → REFUSED (exit 3), nothing written
verify the sealed ledgercopy
# auditor, verify-only IAM
redteam-verify audit/ENG-2026-04-001.jsonl \
  audit/ENG-2026-04-001.seal --hmac-key-file key
# OK: chain intact · OK: HMAC seal verified

Anatomy of an engagement

One YAML roots every run — scope and intent versioned together. Hover a highlighted field to see what it controls.

# engagements/example.yaml id: ENG-2026-04-001 operator: alice@example.com # signer principal authorized_by: ciso@example.com window: { start: …T09:00Z, end: …T17:00Z } scope: targets: [https://staging.example.com, 10.20.0.0/24] out_of_scope: […/admin] # deny wins egress_allowlist: [api.anthropic.com, …] assets: # optional whitebox surface source_repos: [{ path: ./targets/example-api }] budget: { max_turns: 200, max_usd: 25.00, max_tool_calls_per_target: 50 } tools: [recon, web, whitebox, report] subagents: [recon, analyst, whitebox] external_mcp: [] # {atlassian} only objective: | Identify unauthenticated endpoints exposing PII. Do NOT attempt write/delete. Confirm twice.
Hover a field →Each highlighted line in the spec maps to a runtime control the hooks consult on every tool call.
idStable engagement identifier (^[A-Z0-9][A-Z0-9\-_]{2,63}$). Names the ledger and SARIF outputs.
operatorThe declared signer. Signature verification binds the SSH principal to this exact address — another authorized principal's signature is rejected.
windowA hard time bound. The engagement is only authorized between start and end.
scopeThe heart of the guard. out_of_scope is checked first (deny-wins); a call must then match a target; URL hosts are checked against egress.
assetsSource / IaC / specs mounted read-only. Feeds the whitebox pack so the agent gets a far stronger surface than blackbox alone.
budgetTurn, USD, and per-target call caps. The budget gate runs before the scope guard on every PreToolUse.
toolsThe packs to load — everything else is unreachable. whitebox only loads when assets: is non-empty.
subagentsWhich subagents the parent may spawn via the Task tool, each with its own tool subset.
external_mcpHard allowlist of exactly {atlassian}. GitHub is reached via the gh CLI, not an MCP. A second provider is a deliberate code change.
objectiveThe natural-language brief, combined with org-wide rules of engagement into the system prompt. Versioned alongside the scope that bounds it.
